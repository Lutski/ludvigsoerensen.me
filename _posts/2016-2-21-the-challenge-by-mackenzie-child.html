Introduction

On the 21 of February 2016, did one of my favourite youtubers Mackenzie Child start (and complete) a 24 hour design, code and launch challenge (Here is the video). He created the color palette website lolcolors.com. That inspired me to do my own 24 hour design, code and launch challenge. Here I've written a small post about it.

The idea

I have been wanting to build a small application for a while now, but I never did anything. The application I wanted to build was a very simple Winelog applicaiton. The user should have the ability to keep track of the wines they drink, while simultaneously discover new wines.

Wireframes

Before each web, project most people (I think) draw out a few sketches and wireframes. Well so did I, here is a little "image collage" of a few wireframes I did before diving into the designing and coding part of the challenge.

Coding

This is where I spend most of my time. The project was build in Ruby on Rails and you can see the code on Github, I wanted to implement a user authentication system, and a basic CRUD with image uploading for the wines.

Conclusion

At around 3-4am I pushed the last changes to Github finished the challenge a few hours before time was up. This was a really great and fun challenge, it was a way to just get started instead of "doing nothing". If you are reading this I highly recommend that you try out this challenge for yourself, and at the end of the challenge you'll be left with a MVP (Minimum Viable Product) that you can expand on in the future.
